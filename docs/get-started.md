# Get Started with Generative Sites

Personalized content. Your design. This guide walks you through every path — from a 2-minute zero-code setup to a full ABM campaign with baked URLs.

---

## Prerequisites

- A [Personize](https://personize.com) account
- Your Personize secret key (`sk_live_...`) — found in your dashboard under API Keys

---

## Step 0: Generate Your Site Key

Your public key (`pk_live_...`) is derived from your Personize secret key. No new backend — one command:

```bash
npx tsx -e "
  import { generateSiteKey } from '@generative-sites/core';
  const site = generateSiteKey(process.env.PERSONIZE_SECRET_KEY, 'marketing-site');
  console.log('Site ID:    ', site.siteId);
  console.log('Public key: ', site.publicKey);
  console.log('Test key:   ', site.testKey);
"
```

Output:

```
Site ID:     marketing-site
Public key:  pk_live_marketing-site_a3f8x1     ← put this in your script tag
Test key:    pk_test_marketing-site_h7m2p9     ← use for development/preview
```

The public key is **safe to put in client-side code** — it's not your Personize API key. It's a site identifier with a cryptographic suffix that proves it was generated by someone with the secret key. The Edge API validates it by re-deriving the HMAC. No database needed.

For multiple sites, run once per site:

```bash
# Marketing site
generateSiteKey('sk_live_...', 'marketing-site')  → pk_live_marketing-site_a3f8x1

# Documentation
generateSiteKey('sk_live_...', 'docs')             → pk_live_docs_m3p8q1

# Client portal
generateSiteKey('sk_live_...', 'client-portal')    → pk_live_client-portal_j9w2k5
```

## Path 1: Zero-Code (2 minutes)

No npm. No build step. No repo. Just a script tag and HTML attributes.

### Step 1: Add the script tag

Paste this in your website's `<head>` (or before `</body>`):

```html
<script
  src="https://cdn.personize.com/gs.js"
  data-key="pk_live_YOUR_KEY"
  async
></script>
```

This works on any CMS — WordPress, Webflow, Shopify, Squarespace, static HTML, anything. See [cms-guides.md](cms-guides.md) for platform-specific instructions.

### Step 2: Add zones to your text elements

Add `data-gs-zone` to any text element you want personalized:

```html
<h1 data-gs-zone="headline">Welcome to our platform</h1>
<p data-gs-zone="subheadline">We help teams build faster.</p>
<a href="/demo">
  <span data-gs-zone="cta-text">Book a Demo</span>
</a>
<p data-gs-zone="proof">Trusted by 500+ companies worldwide.</p>
```

### Step 3: Done

Visit your page. gs.js will:
1. Discover all `data-gs-zone` elements
2. Detect the visitor's location from edge headers (free, automatic)
3. Open an SSE connection to generate personalized text
4. Replace the text content in each zone with a smooth fade transition

A visitor from Austin, TX might see:

```
headline:    "Trusted by teams across Texas"
subheadline: "The platform Austin's fastest-growing companies rely on"
cta-text:    "Start free trial"
proof:       "14 companies in Austin use our platform daily"
```

The original text is the SEO fallback — Google indexes that. Personalized text is for human visitors.

### What just happened

```
Browser                     GS Edge API                  Personize
───────                     ───────────                  ─────────
gs.js loads
  discovers 4 zones
  opens SSE connection  ──► /api/gs/stream
                            resolves location: Austin,TX
                            calls smartGuidelines()  ──► brand voice rules
                            calls prompt()           ──► AI generates text
                        ◄── streams zone events (SSE)
  replaces textContent
  smooth opacity fade
```

## Path 2: Property Zones — Content from Personize Memory (No LLM)

If you already store data in Personize (via agents, pipelines, MCP, Zapier, or any integration), you can display it directly on your website without any AI generation at serve time.

### Step 1: Store content in Personize

Use any method to populate a contact's properties:

```typescript
// Via Personize SDK (in a pipeline, agent, or script)
import { Personize } from '@personize/sdk';
const personize = new Personize({ secretKey: 'sk_live_...' });

await personize.memory.memorize({
  email: 'sarah@acme.com',
  content: `
    Hero headline: How Acme Corp scaled API operations by 3x
    Meeting summary: Discussed migration timeline, agreed on Q2 kickoff
    Next steps: Schedule technical deep-dive with engineering team
  `,
  collection: 'client_portal',
  enhanced: true,  // AI extracts into structured properties
});
```

Or via Zapier, n8n, MCP, or any tool that calls the Personize API.

### Step 2: Reference properties in your HTML

Use the `collection:property` syntax in your zone attribute:

```html
<h1 data-gs-zone="client_portal:hero_headline">Welcome back</h1>
<div data-gs-zone="client_portal:meeting_summary">Your meeting notes will appear here.</div>
<p data-gs-zone="client_portal:next_steps">We'll follow up with next steps.</p>
```

### Step 3: Identify the visitor

The visitor needs to be identified so gs.js knows which contact's properties to fetch. Three ways:

**Option A: Auth session (SaaS apps)**

```html
<script>
  window.__GS_USER__ = {
    email: 'sarah@acme.com',
    firstName: 'Sarah',
    company: 'Acme Corp'
  };
</script>
<script src="https://cdn.personize.com/gs.js" data-key="pk_live_..." async></script>
```

**Option B: Unique URL (campaigns)**

Send Sarah a link: `https://yoursite.com/for/sarah-chen-acme`

The slug resolves to her email via Personize memory lookup.

**Option C: URL token (email links)**

Send Sarah any link with an encrypted token: `https://yoursite.com/portal?gs=eyJ1c2Vy...`

The token decrypts to her email server-side.

### What just happened

```
Browser                     GS Edge API                  Personize
───────                     ───────────                  ─────────
gs.js loads
  discovers 3 property zones
  opens SSE connection  ──► /api/gs/stream
                            resolves identity: sarah@acme.com
                            property zones:
                              recall('client_portal',  ──► returns stored value
                                'hero_headline')            instantly, no LLM
                        ◄── streams zone events (SSE)
  replaces textContent
```

No LLM call. No generation cost. Instant. Deterministic.

## Path 3: Mixing Generative + Property Zones

You can use both zone types on the same page. They resolve in parallel:

```html
<!-- Property zone: reads from Personize memory (instant, no LLM) -->
<h1 data-gs-zone="client_portal:hero_headline">Welcome back</h1>

<!-- Property zone: reads from Personize memory -->
<div data-gs-zone="client_portal:meeting_summary">Meeting notes here</div>

<!-- Generative zone: AI-written at serve time using contact context -->
<p data-gs-zone="recommendation">Check out our latest features.</p>

<!-- Generative zone: AI-written -->
<span data-gs-zone="cta-text">Get Started</span>
```

Property zones cost zero LLM calls. Generative zones cost one `prompt()` call total (all generative zones on the page are batched into a single request).

## Path 4: ABM Campaign with Unique URLs

Generate personalized pages for a list of contacts, each with their own URL.

### Step 1: Install dependencies

```bash
npm install @generative-sites/core @personize/sdk dotenv tsx
```

### Step 2: Create your contact list and bake script

```typescript
// bake-campaign.ts
import 'dotenv/config';
import { Personize } from '@personize/sdk';
import { generateSlug, generateContactToken } from '@generative-sites/core';
import type { CampaignContact } from '@generative-sites/core';

const personize = new Personize({ secretKey: process.env.PERSONIZE_SECRET_KEY! });

// Your ABM target list
const contacts: CampaignContact[] = [
  { email: 'sarah@acme.com', firstName: 'Sarah', lastName: 'Chen', company: 'Acme Corp' },
  { email: 'marcus@techcorp.io', firstName: 'Marcus', lastName: 'Rivera', company: 'TechCorp' },
  { email: 'lisa@fintech.co', firstName: 'Lisa', lastName: 'Park', company: 'FinTech Solutions' },
];

// Zones to personalize
const zones = [
  // Property zones: pulled from Personize memory (no LLM)
  { id: 'website_zones:hero_headline' },
  { id: 'website_zones:sub_headline' },

  // Generative zones: AI-written using contact context
  { id: 'proof', prompt: 'Write social proof relevant to their industry, max 2 sentences.' },
  { id: 'cta-text', prompt: 'Write a CTA with their first name. Max 5 words.' },
];

async function main() {
  for (const contact of contacts) {
    const slug = generateSlug(contact);
    const token = generateContactToken(contact.email, process.env.GS_TOKEN_SECRET!);

    console.log(`${contact.email}`);
    console.log(`  Unique URL: https://yoursite.com/for/${slug}`);
    console.log(`  Token URL:  https://yoursite.com/pricing?gs=${token}`);
    console.log('');

    // Store the slug mapping so the edge API can resolve it
    await personize.memory.memorize({
      email: contact.email,
      content: `Website personalization slug: ${slug}`,
      collection: 'website_zones',
      enhanced: true,
      tags: ['generative-sites', 'slug-mapping'],
    });
  }

  console.log('Done! Send these URLs in your email campaign.');
}

main().catch(console.error);
```

### Step 3: Create the `.env` file

```bash
PERSONIZE_SECRET_KEY=sk_live_your_key_here
GS_TOKEN_SECRET=your_256_bit_secret_here
```

Generate a token secret:

```bash
npx tsx -e "import {generateTokenSecret} from '@generative-sites/core'; console.log(generateTokenSecret())"
```

### Step 4: Run the bake script

```bash
npx tsx bake-campaign.ts
```

Output:

```
sarah@acme.com
  Unique URL: https://yoursite.com/for/sarah-chen-acme
  Token URL:  https://yoursite.com/pricing?gs=eyJ1c2VyI...

marcus@techcorp.io
  Unique URL: https://yoursite.com/for/marcus-rivera-techcorp
  Token URL:  https://yoursite.com/pricing?gs=bWFyY3Vz...

lisa@fintech.co
  Unique URL: https://yoursite.com/for/lisa-park-fintech-solutions
  Token URL:  https://yoursite.com/pricing?gs=bGlzYUBm...

Done! Send these URLs in your email campaign.
```

### Step 5: Create your landing page

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <title>Your Platform — Personalized</title>
  <script src="https://cdn.personize.com/gs.js" data-key="pk_live_..." async></script>
</head>
<body>
  <h1 data-gs-zone="website_zones:hero_headline">See how we can help your team</h1>
  <p data-gs-zone="website_zones:sub_headline">The modern platform for growing teams</p>
  <a href="/demo"><span data-gs-zone="cta-text">Book a Demo</span></a>
  <p data-gs-zone="proof">Trusted by 500+ companies worldwide.</p>
</body>
</html>
```

### Step 6: Send the URLs

Import the URLs into your email tool (HubSpot, Outreach, Salesloft, or any tool):

```

Take a look — I think you'll find it relevant.
```

When Sarah clicks, she sees content written for her. Marcus sees his version. Lisa sees hers. Same page design, different words.

## Path 5: CRM Sync (HubSpot)

Sync contacts from HubSpot to Personize, generate URLs, and push engagement data back.

### Step 1: Install

```bash
npm install @generative-sites/core @hubspot/api-client @personize/sdk dotenv tsx
```

### Step 2: Sync script

```typescript
// hubspot-sync.ts
import 'dotenv/config';
import { HubSpotAdapter } from './adapters/crm/hubspot/src/index';

const adapter = new HubSpotAdapter({
  hubspotToken: process.env.HUBSPOT_TOKEN!,
  personizeSecretKey: process.env.PERSONIZE_SECRET_KEY!,
  tokenSecret: process.env.GS_TOKEN_SECRET,
  siteBaseUrl: 'https://yoursite.com',
});

async function main() {
  // 1. Sync contacts → Personize memory
  const sync = await adapter.syncContactsToPersonize(100);
  console.log(`Synced ${sync.synced} contacts to Personize`);

  // 2. Generate URLs + tokens → store in HubSpot contact properties
  const urls = await adapter.generateURLsForContacts();
  console.log(`Generated URLs for ${urls.updated} contacts`);

  // Now in HubSpot, each contact has:
  //   gs_personalized_url = "https://yoursite.com/for/sarah-chen-acme"
  //   gs_token = "eyJ1c2VyIjoic2FyYWh..."
  //
  // Use in email templates:
  //   <a href="{{contact.gs_personalized_url}}">See your page</a>
}

main().catch(console.error);
```

### Step 3: `.env`

```bash
PERSONIZE_SECRET_KEY=sk_live_your_key_here
HUBSPOT_TOKEN=pat-na1-your_hubspot_token
GS_TOKEN_SECRET=your_256_bit_secret_here
```

### Step 4: Run

```bash
npx tsx hubspot-sync.ts
```

Schedule this as a nightly cron or trigger it via webhook when contacts are updated.

## Path 6: Memorize from Website (Capture Visitor Input)

Capture content from your website and write it back to Personize as structured properties. Off by default, opt-in per element.

### Capture a single field

```html
<label>What's your main goal?</label>
<textarea
  data-gs-memorize="onboarding:primary_goal"
  data-gs-trigger="blur"
  placeholder="e.g., Reduce API response times"
></textarea>
```

When the visitor types and tabs away, the value is sent to Personize and stored as the `primary_goal` property in the `onboarding` collection for that contact.

### Capture a form

```html
<form data-gs-memorize-form="feedback">
  <input
    name="feature_request"
    data-gs-memorize="feedback:feature_request"
    placeholder="What feature would help you most?"
  />
  <select data-gs-memorize="feedback:satisfaction" data-gs-trigger="change">
    <option value="">How satisfied are you?</option>
    <option value="very_satisfied">Very satisfied</option>
    <option value="satisfied">Satisfied</option>
    <option value="neutral">Neutral</option>
    <option value="unsatisfied">Unsatisfied</option>
  </select>
  <textarea
    data-gs-memorize="feedback:comments"
    placeholder="Any additional comments?"
  ></textarea>
  <button type="submit">Submit Feedback</button>
</form>
```

On submit, all `data-gs-memorize` fields in the form are captured and written to Personize in one batch.

### Capture via JavaScript

```javascript
// After any interaction
GS.memorize('client_portal:last_action', 'Reviewed Q2 proposal');
GS.memorize('preferences:timezone', Intl.DateTimeFormat().resolvedOptions().timeZone);
```

### Important: visitor must be identified

Memorize writes require a known visitor. The visitor must be identified via one of:

- `window.__GS_USER__` (auth session)
- `?gs=token` (URL token)
- `/for/slug` (unique URL)
- `GS.identify('email@company.com')` (explicit call)

Anonymous visitors' memorize writes are silently dropped — no errors, no broken UX.

### Round-trip: write on one page, read on another

```
Page A (feedback form):
  <textarea data-gs-memorize="feedback:feature_request">
  → Visitor types: "Need better batch processing"
  → Stored in Personize: feedback.feature_request = "Need better batch processing"

Page B (dashboard, next visit):
  <p data-gs-zone="feedback:feature_request">Tell us what you need</p>
  → Shows: "Need better batch processing"
```

## Path 7: SaaS Dashboard (Auth Session)

For logged-in applications where you know the user.

### Step 1: Inject user identity before gs.js loads

```tsx
// Next.js layout.tsx (or _app.tsx, or any framework)
export default function Layout({ children }) {
  const user = useCurrentUser(); // your auth hook

  return (
    <html>
      <head>
        {user && (
          <script
            dangerouslySetInnerHTML={{
              __html: `window.__GS_USER__ = ${JSON.stringify({
                email: user.email,
                firstName: user.firstName,
                company: user.company,
                plan: user.plan,
                role: user.role,
              })};`,
            }}
          />
        )}
        <script
          src="https://cdn.personize.com/gs.js"
          data-key="pk_live_..."
          async
        />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

### Step 2: Add zones to your dashboard

```tsx
export default function Dashboard() {
  return (
    <div>
      <h1 data-gs-zone="dashboard-greeting">Welcome back</h1>
      <p data-gs-zone="usage-insight">Here's your usage summary.</p>
      <div data-gs-zone="recommendation">Check out our latest features.</div>
    </div>
  );
}
```

### Step 3: Track product events for better personalization

```tsx
// Track feature usage
function onFeatureClick(featureName: string) {
  GS.track('feature_used', { feature: featureName });
}

// Track upgrade interest
function onPricingView(plan: string) {
  GS.track('pricing_viewed', { plan });
}
```

## API Reference

### gs.js (browser)

```javascript
// Identify the visitor (after login or form fill)
GS.identify('sarah@acme.com', { firstName: 'Sarah', company: 'Acme Corp' });

// Track custom events (batched, sent every 5s)
GS.track('feature_used', { feature: 'batch-api' });

// Write to a specific collection:property
GS.memorize('feedback:feature_request', 'Need better batch processing');

// Listen for events
GS.on('zone:render', function(data) { console.log(data.zone, data.text); });
GS.on('meta', function(data) { console.log(data.tier, data.location); });
GS.on('memorize', function(data) { console.log(data.target, data.value); });

// Force re-render zones
GS.refresh();           // all zones
GS.refresh('headline'); // specific zone

// Debug
console.log(GS.debug());
```

### HTML Attributes

```html
<!-- READ: display personalized content -->
<h1 data-gs-zone="headline">Fallback text</h1>
```
